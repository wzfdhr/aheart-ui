# InputNumber Invalid Boundary Controls Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Prevent invalid or empty current values from being treated as min/max boundaries for `AInputNumber` step-control disabled states.

**Architecture:** Keep the existing inclusive min/max disabled computeds, but guard them with the same current-value validity helper used by ARIA and state-class logic. This keeps the behavior local to button disabled state and avoids changing parser, clamp, or step emission behavior.

**Tech Stack:** Vue 3 `<script setup>`, TypeScript, Vitest with Vue Test Utils, Vite build output, VitePress docs.

### Task 2: Guard Boundary Disabled Computeds

**Files:**
- Modify: `packages/components/src/input-number/input-number.vue`

- [x] **Step 1: Guard max-boundary comparison**

Update `isValueAtOrAboveMax`:

```ts
const isValueAtOrAboveMax = computed(
  () =>
    props.max !== undefined &&
    mergedValue.value !== undefined &&
    mergedValue.value !== '' &&
    isValidValueString(String(mergedValue.value)) &&
    compareDecimalStrings(String(mergedValue.value), String(props.max)) >= 0
)
```

- [x] **Step 2: Guard min-boundary comparison**

Update `isValueAtOrBelowMin`:

```ts
const isValueAtOrBelowMin = computed(
  () =>
    props.min !== undefined &&
    mergedValue.value !== undefined &&
    mergedValue.value !== '' &&
    isValidValueString(String(mergedValue.value)) &&
    compareDecimalStrings(String(mergedValue.value), String(props.min)) <= 0
)
```
