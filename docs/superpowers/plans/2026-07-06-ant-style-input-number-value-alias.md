# InputNumber Value Alias Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Allow `AInputNumber` to accept `value` as a controlled value alias while preserving `modelValue` as the preferred Vue API.

**Architecture:** The component adds a `value` prop and updates the controlled/merged value computed properties. `modelValue` takes precedence over `value`, and existing events remain unchanged.

**Tech Stack:** Vue 3 script setup, TypeScript, Vitest, VitePress, existing aheart-ui component build.
