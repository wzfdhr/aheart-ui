# Runtime Internationalization Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Chinese-default runtime locale packs and allow ConfigProvider to localize Empty, Pagination, Table, and Modal.

**Architecture:** Keep locale data in the existing config module as typed plain objects. ConfigProvider continues to inject a computed merged config, but merges each supported locale namespace so nested providers can override one string without discarding sibling strings. Components consume only their own locale namespace and retain explicit props and render callbacks as higher-precedence overrides.

**Tech Stack:** Vue 3, TypeScript, Vitest, Vue Test Utils, VitePress.

## Global Constraints

- Keep Vue peer range `>=3.4.0 <4` and add no i18n or date dependencies.
- Export `zhCN`, `enUS`, and `AheartLocale` from the root `aheart-ui` entry point.
- Default locale is Simplified Chinese.
- Locale packs must be SSR-safe plain data without browser globals.
- Explicit component props and existing render callbacks remain higher precedence than locale defaults.
- Generated `es` and `lib` artifacts are refreshed only after source tests and type checks pass.
