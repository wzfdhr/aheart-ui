# Radio AutoFocus Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add `autoFocus` support to `ARadio`, focusing the native radio input when the component mounts.

**Architecture:** Reuse the existing `focus()` helper so imperative and mount-time focus target the same native input. Add only one prop and one mounted hook; leave group behavior and event payloads unchanged.

**Tech Stack:** Vue 3 `<script setup>`, TypeScript, Vitest with Vue Test Utils, Vite build output, VitePress docs.

- [x] **Step 1: Write the failing test**

Add this test near the existing focus/blur methods test:

```ts
it('focuses the native input when autoFocus is enabled', async () => {
  const host = document.createElement('div')
  document.body.appendChild(host)

  const wrapper = mount(Radio, {
    attachTo: host,
    props: { autoFocus: true }
  })

  await nextTick()

  expect(document.activeElement).toBe(wrapper.find('input').element)

  wrapper.unmount()
  host.remove()
})
```
