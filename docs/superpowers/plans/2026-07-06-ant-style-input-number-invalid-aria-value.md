# InputNumber Invalid ARIA Value Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Omit `aria-valuenow` when the current `AInputNumber` value is non-empty but not a valid finite decimal.

**Architecture:** Keep the existing `ariaValueNow` computed as the single template binding. Extend it to reuse `isValidValueString`, so the accessibility value follows the same decimal validity rules already used by parser and default display helpers.

**Tech Stack:** Vue 3 `<script setup>`, TypeScript, Vitest with Vue Test Utils, VitePress docs.

### Task 2: Implement Invalid ARIA Guard

**Files:**
- Modify: `packages/components/src/input-number/input-number.vue`

- [x] **Step 1: Update `ariaValueNow`**

Replace the computed with:

```ts
const ariaValueNow = computed(() => {
  if (mergedValue.value === undefined || mergedValue.value === '') {
    return undefined
  }

  const value = String(mergedValue.value)

  return isValidValueString(value) ? value : undefined
})
```
