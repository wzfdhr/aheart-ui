# InputNumber String Step Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Allow `AInputNumber` to accept `step` as `number | string` and treat string steps as numeric offsets for component interactions.

**Architecture:** The prop type widens in `types.ts`. `input-number.vue` adds a small numeric normalizer and uses it for controls, keyboard, and wheel stepping while keeping the native `step` attribute unchanged.

**Tech Stack:** Vue 3, TypeScript, Vitest, VitePress, existing aheart-ui component build.

- [x] **Step 1: Widen the prop type**

Change `step` to accept `[Number, String]` with `PropType<number | string>`.

- [x] **Step 2: Normalize step for interactions**

Add a computed numeric step value:

```ts
const resolvedStep = computed(() => {
  const value = Number(props.step)
  return Number.isFinite(value) ? value : 1
})
```

Use `resolvedStep.value` for click, keyboard, and wheel offsets.
