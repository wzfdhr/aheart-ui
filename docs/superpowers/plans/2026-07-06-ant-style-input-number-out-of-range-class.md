> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an `is-out-of-range` root state class to `AInputNumber` for valid current values outside `min`/`max`.

**Architecture:** Reuse the existing decimal validity and comparison helpers so the new state class follows the same range semantics as the component's current decimal logic. Keep the behavior isolated in one computed used only by the root class object.

**Tech Stack:** Vue 3 `<script setup>`, TypeScript, Vitest with Vue Test Utils, VitePress docs.

### Task 2: Implement Out-Of-Range Root Class

- [x] **Step 1: Add `isOutOfRange` computed**

Add this computed near `isNotANumber`:

```ts
const isOutOfRange = computed(() => {
  if (mergedValue.value === undefined || mergedValue.value === '') {
    return false
  }

  const value = String(mergedValue.value)

  if (!isValidValueString(value)) {
    return false
  }

  return (
    (props.min !== undefined && compareDecimalStrings(value, String(props.min)) < 0) ||
    (props.max !== undefined && compareDecimalStrings(value, String(props.max)) > 0)
  )
})
```
