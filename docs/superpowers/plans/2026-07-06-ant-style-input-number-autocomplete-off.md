# InputNumber Autocomplete Off Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Default the inner `AInputNumber` input to `autocomplete="off"` while preserving explicit native overrides.

**Architecture:** Keep native attr forwarding centralized in `inputAttrs`. Add one computed helper for the autocomplete value and exclude autocomplete keys from the forwarded attrs so the template binding has a single authoritative value.

**Tech Stack:** Vue 3 `<script setup>`, TypeScript, Vitest with Vue Test Utils, VitePress docs.

### Task 2: Implement Autocomplete Default

**Files:**
- Modify: `packages/components/src/input-number/input-number.vue`

- [x] **Step 1: Add computed autocomplete helper**

Add this computed near `inputMode` and `ariaValueNow`:

```ts
const inputAutoComplete = computed(() => {
  const value = attrs.autocomplete ?? attrs.autoComplete

  return typeof value === 'string' ? value : 'off'
})
```

- [x] **Step 2: Bind autocomplete on the native input**

Add this binding to the native `<input>`:

```vue
:autocomplete="inputAutoComplete"
```

- [x] **Step 3: Exclude autocomplete keys from forwarded attrs**

Update the `inputAttrs` filter to exclude `autocomplete` and `autoComplete`:

