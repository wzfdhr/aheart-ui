# InputNumber Default Value Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add `defaultValue` support to `AInputNumber` for uncontrolled initial values.

**Architecture:** The component keeps controlled behavior when `modelValue` is provided. When `modelValue` is absent, a local `uncontrolledValue` ref initialized from `defaultValue` drives display and updates after input/step interactions.

**Tech Stack:** Vue 3 script setup, TypeScript, Vitest, VitePress, existing aheart-ui component build.

- [x] **Step 2: Add uncontrolled state**

Add:

```ts
const uncontrolledValue = ref<number | undefined>(props.defaultValue)
const isControlled = computed(() => props.modelValue !== undefined)
const mergedValue = computed(() => (isControlled.value ? props.modelValue : uncontrolledValue.value))
```

Use `mergedValue` for display and stepping. Update `uncontrolledValue` inside the value commit path only when uncontrolled.
